# Site review: SnappTools vs [OnlineTools.com](https://onlinetools.com/)

Comparison and improvement checklist based on OnlineTools.com (Browserling’s toolkit).

---

## What OnlineTools.com does well (and we can adopt)

### 1. **Clear “What / How / Who” on homepage**
- **What are Online Tools?** – One paragraph explaining the product (digital Swiss Army knife, no install, one-stop shop).
- **How do Online Tools work?** – Input on left, output on right, processing in browser, speed + privacy.
- **Who uses Online Tools?** – Designers, developers, sellers, enterprises, universities, Fortune 100.

**Gap:** We have hero + intro but no dedicated “What / How / Who” blocks.  
**Action:** Add a **What / How / Who** section on the homepage (implemented below).

---

### 2. **Category descriptions**
- Each category (PNG, JPG, Text, etc.) has a short line: “Tools for working with…”, then “See All X Tools” + “Try [Most Popular]”.

**Gap:** We only show category titles (Generator Tools, Developer Tools, etc.), no short description.  
**Action:** Add one-line descriptions per category in ToolsHub.

---

### 3. **Footer: categories + contact + “top tools”**
- Product (Features, Pricing, About Us).
- **Tool categories** – Full list of category names as links.
- **Top tools** – Many tool names linked.
- **Contact** – “Contact Us” + email (e.g. hello@onlinetools.com) visible in footer.

**Gap:** Our footer has Site + Trust & legal but no tool categories list, no visible email, no “request a tool” angle.  
**Action:** Add contact email in footer; add “Tool categories” links; add “Can’t find a tool? Contact us.”

---

### 4. **“Request / feedback” CTA**
- “Didn’t find the tool you were looking for? Tell us what we’re missing” with email + message form.

**Gap:** We have a Contact page but no clear “request a tool” or “suggest a tool” on homepage/footer.  
**Action:** Add a short CTA: “Need a different tool? Contact us” with link to /contact.

---

### 5. **Social proof**
- “Our tools and products are trusted by” + logos.
- Testimonials section.

**Gap:** We have a trust strip (Free, No signup, Private, Secure) but no “trusted by” or testimonials.  
**Action:** Optional later: “Used by developers and small teams” line or placeholder for logos/testimonials.

---

### 6. **Pricing / accounts**
- They have Sign In, Sign Up, Pricing (premium plans).

**Gap:** We are 100% free, no accounts.  
**Action:** None required; our “free, no signup” is a differentiator.

---

### 7. **Search**
- They have a global search for tools.

**Gap:** We have no search.  
**Action:** Optional later: simple client-side filter on tool names on /#tools or /tools page.

---

### 8. **Category landing pages**
- They have /category/png-tools etc. with full category description + tool list.

**Gap:** We have a single /#tools section with categories; no /tools/category/generators URLs.  
**Action:** Optional later: add /tools/category/[category] if we want deeper SEO.

---

## What we already do well (vs OnlineTools)

| Area | Us | Them |
|------|----|------|
| **Blog / guides** | 17+ articles, visible on homepage + tool pages | No blog in the same way |
| **Trust pages** | Privacy, About Us, Contact, Terms in footer | Same (Privacy, Terms, Contact) |
| **Tool page content** | 300–500+ words + related guides per tool | Mostly tool UI |
| **Free / no signup** | Everything free, no account | Free tier + premium |
| **Internal linking** | Tools ↔ blog, related tools, related guides | Category/tool links |
| **Homepage SEO** | Hero + intro + guides section + FAQ | What/How/Who + categories |

---

## Priority actions (implemented in code)

1. Add **What / How / Who** section on homepage.
2. Add **short category descriptions** in ToolsHub.
3. Show **contact email** in footer and add **“Tool categories”** links.
4. Add **“Can’t find a tool? Contact us”** CTA (e.g. after tools or in footer).

---

## Optional later

- Testimonials or “Used by” line.
- Tool search or filter on homepage.
- Category landing pages (/tools/category/generators etc.).
- Newsletter signup (like their “Subscribe to our updates”).
